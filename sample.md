# GitHub Pages Jekyll 블로그의 카테고리 관리 방식

조사 결과를 정리해드리겠습니다.

---

## 1. 카테고리 페이지 구현 방식

### **A. 카테고리별 개별 페이지 생성 (가장 일반적)**

루트 디렉토리에 `_category` 디렉토리를 만들고, 지원하려는 각 카테고리마다 파일을 넣습니다. 예를 들어 android.md, golang.md, java.md 등을 만듭니다.

각 파일은 최소한의 front matter만 포함하고, 공통 레이아웃을 통해 해당 카테고리의 글 목록을 렌더링합니다.

**장점:** URL이 깔끔함 (`/category/jekyll/`)
**단점:** 카테고리 추가할 때마다 파일을 수동 생성해야 함

### **B. 단일 "전체 카테고리" 페이지**

일반적인 블로그는 독자가 카테고리별 또는 날짜별로 글을 볼 수 있는 방법을 제공하여, 블로그 포스트를 하나씩 훑어보지 않아도 되게 합니다.

`/categories/` 같은 단일 페이지에서:
- 모든 카테고리명과 글 개수 표시
- 각 카테고리 아래에 **글 제목만** 나열 (본문 미리보기 없음)
- 앵커 링크로 빠른 이동 가능

## 2. 날짜별 아카이브 페이지

### **연도/월별 그룹핑**

Jekyll은 `group_by_exp` 필터를 추가했는데, 이걸 쓰면 훨씬 깔끔하게 구현할 수 있습니다:

```liquid
{% assign postsByYearMonth = site.posts | group_by_exp:"post", "post.date | date: '%Y년 %m월'" %}
{% for yearMonth in postsByYearMonth %}
  <h3>{{ yearMonth.name }}</h3>
  <ul>
  {% for post in yearMonth.items %}
    <li><a href="{{ post.url }}">{{ post.title }}</a></li>
  {% endfor %}
  </ul>
{% endfor %}
```

**결과물 예시:**
```
## 2024년 12월
- 글 제목 1
- 글 제목 2

## 2024년 11월
- 글 제목 3
```

이렇게 하면 **글 본문 없이 제목 목록만** 보여줄 수 있습니다 — 질문하신 것과 정확히 일치하는 방식이에요.

## 3. 다중 카테고리 처리

### **하나의 글이 여러 카테고리에 속할 때**

`_posts` 위의 모든 디렉토리는 카테고리로 읽힙니다. 카테고리와 태그의 핵심적인 차이는 카테고리는 포스트의 URL에 포함될 수 있지만, 태그는 그렇지 않다는 것입니다.

**해결책 1: JavaScript 필터링**

각 카테고리별로 버튼(또는 텍스트 링크)을 렌더링합니다. 버튼 클릭 시 모든 사이트 포스트를 순회하는 함수를 호출합니다. 클릭한 카테고리가 해당 포스트의 카테고리 목록에 있으면 표시하고, 아니면 숨깁니다.

→ 페이지 새로고침 없이 실시간 필터링 가능

**해결책 2: 중복 표시**
- 글이 `[개발, Python]` 두 카테고리에 속하면
- `/category/개발/`과 `/category/Python/` 둘 다에 표시됨

## 4. 네비게이션 구조

### **일반적인 네비게이션 설정**

`_data/navigation.yml`에서 메인 키 아래에 제목과 URL을 추가하여 네비게이션 링크를 정의합니다:

```yaml
main:
  - title: "글 목록"
    url: /archives/
  - title: "카테고리"
    url: /categories/
  - title: "태그"
    url: /tags/
```

### **플러그인 사용: jekyll-archives**

날짜, 태그, 카테고리별로 포스트 아카이브를 자동 생성합니다. Jekyll 설정 파일의 jekyll-archives 키로 설정할 수 있습니다.

자동 생성되는 페이지들:
- `/archives/year/2024/`
- `/archives/month/2024-01/`
- `/category/jekyll/`

⚠️ **주의:** GitHub Pages는 커스텀 플러그인을 지원하지 않아서, `jekyll-archives`를 쓰려면 로컬 빌드 후 배포하거나 GitHub Actions를 사용해야 합니다.

## 5. 글 제목 리스트 스타일 (질문하신 방식)

대부분의 잘 정리된 Jekyll 블로그는 **아카이브/인덱스 페이지**를 따로 만들어서:
- 글 제목 (링크)
- 날짜 (짧은 포맷)
- 선택적으로: 카테고리 태그

이런 식으로 표시합니다:

```
## 2024
- 12월 15일 — 첫 번째 글 제목
- 11월 22일 — 두 번째 글 제목
- 10월 03일 — 세 번째 글 제목

## 2023
- ...
```

해당 카테고리의 모든 포스트를 순회하면서, 각 블로그 포스트로 연결되는 포스트 제목을 출력합니다.

전체 excerpt를 보여주는 것보다 이 방식이 훨씬 깔끔합니다.

## 6. 정리: 추천 구현 방식

| 필요한 기능 | 해결책 |
|------------|--------|
| 카테고리별 탐색 | `/categories/` 페이지 만들고, 각 카테고리 아래 글 제목만 나열 |
| 날짜별 탐색 | `/archives/` 페이지에서 `group_by_exp`로 연도/월별 그룹핑 |
| 다중 카테고리 글 | JavaScript 필터링 또는 각 카테고리에 중복 표시 |
| 빠른 네비게이션 | 사이드바에 카테고리 목록(글 개수 포함) + 최근 글 표시 |

---

## 7. 실제 적용 팁

**GitHub Pages에서 플러그인 없이 구현하려면:**

1. `_data/categories.yml`에 카테고리 목록 정의
2. `_layouts/category.html` 레이아웃 생성
3. 각 카테고리마다 `category/카테고리명.md` 파일 생성 (자동화 불가, 수동)
4. `/archives.md` 페이지에서 `group_by_exp`로 날짜별 목록 생성

**이미 잘 구현된 테마 사용 추천:**
- **Minimal Mistakes** — 카테고리, 태그, 아카이브 페이지 기본 제공
- **Chirpy** — 사이드바에 카테고리/태그 자동 표시
- **Just the Docs** — 문서 스타일 네비게이션

이런 테마들은 질문하신 기능들이 이미 구현되어 있어서 커스터마이징만 하면 됩니다.
